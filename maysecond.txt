## The LLM-First Problem

### What happens today

In simplify_to_grade() (simplifier.py:589-621), the very first thing the method does after measuring the source grade is:

    if self.llm_client and direction != 0:        # line 604
        # ... call _llm_full_rewrite() immediately

If FIREWORKS_API_KEY is set in the environment (line 510), self.llm_client is always truthy. So the flow is:

    simplify_to_grade()
      ├─ self.llm_client exists? ──YES──> _llm_full_rewrite()     [LINE 604-621]
      │                                      ├─ Fireworks API call (pass 1)
      │                                      ├─ _rule_correct() (patch attempt)
      │                                      ├─ Still >2 grades off? → LLM correction pass
      │                                      └─ Return LLM text
      │                                   _finalize_preview_candidate()            [LINE 626]
      │                                   Done. Rule-based path NEVER runs.
      │
      └─ self.llm_client is None ──NO──> _select_rewrite_candidate()  [LINE 651]
                                            ├─ 3-stage beam search (lexical→syntactic→discourse)
                                            ├─ _iterative_rule_rewrite()
                                            ├─ _rank_candidates() with scoring
                                            ├─ critic review + repair pass
                                            └─ Return best candidate

The entire rule-based pipeline — the beam search, candidate ranking, scoring, critic review — only ever runs if the Fireworks API key is missing. It's a fallback, not a primary path.

### Why this is a problem

1. Non-deterministic results. The LLM produces different output on different calls, even at temperature=0. Your matrix results change between runs.

2. No candidate selection. The rule-based path generates multiple candidates via beam search, scores them on grade distance + grammar validity + semantic similarity, and picks the best. The LLM path generates exactly one candidate and takes it.

3. The LLM overshoots systematically. Grade 6 → Grade 11 lands at 14.63, Grade 7 → Grade 12 lands at 15.37.

4. Stepping makes it worse. For grade distances > 4, the code does a two-step rewrite via a midpoint. Each step is an independent LLM call, and errors compound.

5. The rule-based path is actually better-engineered. It has beam search, multi-stage candidate generation, a scoring function with grammar and semantic checks, critic review, and repair passes. But none of that runs when the LLM key is present.

### Proposed fix: Rule-first, LLM-as-polish

Flip the architecture so the rule-based beam search always runs first, and the LLM is used only to polish the best rule-based candidate if it's still not close enough:

    def simplify_to_grade(self, text, target_grade, mode='auto'):
        source_grade, _, _ = self._measure_text_metrics(text)
        direction = self._get_target_direction(source_grade, target_grade)
        going_up = direction > 0

        # ALWAYS run the rule-based beam search first
        selection = self._select_rewrite_candidate(text, target_grade, mode)
        current_text = selection['text']
        candidate_score = selection['score']

        # Only use LLM as a polish pass if rule-based result
        # is still >1.5 grades from target
        rule_grade, _, _ = self._measure_text_metrics(current_text)
        if self.llm_client and direction != 0 and abs(rule_grade - target_grade) > 1.5:
            llm_text, _ = self._llm_full_rewrite(current_text, target_grade, going_up=going_up)
            if llm_text:
                llm_grade, _, _ = self._measure_text_metrics(llm_text)
                # Only accept the LLM output if it's actually closer to target
                if abs(llm_grade - target_grade) < abs(rule_grade - target_grade):
                    current_text = llm_text

        # Continue with finalization, diffing, validation as before...

Key changes:
- Rule-based always runs — deterministic, scored candidates every time
- LLM only fires when rules fell short (>1.5 grades off) — not as default
- LLM input is the rule-based output, not the raw original — polishing a near-miss
- LLM output competes — only accepted if it actually beats the rule-based result
