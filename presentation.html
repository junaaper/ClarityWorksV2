Motivation

The Problem

Readability Matters

36% of adults read at or below a basic literacy level. Mismatched reading material leads to poor comprehension, disengagement, and inequitable access to information.

Existing Tools Fall Short

Traditional formulas like Flesch-Kincaid use surface metrics only. They cannot detect semantic complexity, passive voice density, or academic vocabulary.

No End-to-End Solution

Current tools either analyse or simplify — never both. Teachers, editors, and researchers need a single platform for analysis, rewriting, and reference material search.

Our Goal

Build a unified platform that analyses, rewrites, and augments text readability using ML, NLP, and Generative AI.

Core Feature

Bidirectional Text Rewriting Engine

Rewrite in Any Direction

Simplify text down to Grade 3 or upgrade up to College level. The engine auto-detects direction based on current vs. target grade.

Multi-Round Beam Search
  • Beam width of 3 — tracks top 3 candidates
  • Rounds: lexical → split/combine → discourse
  • Grade-specific policy buckets control intensity
  • Scoring: syllables/word + words/sentence distance
Synonym Cascade (5 Sources)
  1. Curated simplification map (90+ pairs)
  2. WordNet + Lesk WSD (context-aware)
  3. Datamuse API (frequency-ranked)
  4. LLM rescue (Llama 3.3 70B)
  5. Complexification map (42 pairs, for upgrades)
LLM Validation & Repair (3-Pass)
Pass 1: Validate Pass 2: Critic Review Pass 3: Polish & Fix

Core Feature

Concept Prerequisite Graphs

What knowledge does a reader need?

Concept graphs map the prerequisite knowledge structure of any text. They show what a reader must already understand before the main ideas will make sense.

Extraction Pipeline
  1. spaCy noun phrase extraction — ranks top-30 key phrases by frequency
  2. LLM concept mapping — Llama 3.3 70B identifies 5–15 concepts and prerequisite edges
  3. Validation & cleanup — removes unconnected nodes, validates edge references
Three Concept Tiers
Prerequisite
Foundational knowledge
Intermediate
Bridging concepts
Target
Main ideas being taught

Available for both individual analyses and uploaded RAG documents. Interactive drag-and-drop canvas with zoom controls.

Key Design Decisions
  • JSONB columns for flexible nested data (words, sentences, changes, metrics, concept graphs)
  • Cascade deletes from users through all child tables
  • Performance indexes on user_id and created_at DESC
  • ChromaDB as separate vector store for embedding persistence
  • Auto-initialisation on server startup (no manual migration step)

Looking Ahead

Future Work & Extensions

Multi-Language Support

Extend readability analysis beyond English to Spanish, French, Arabic, and other languages. Train language-specific models on equivalent corpora.

Collaborative Editing

Real-time collaborative rewriting for teams of educators. WebSocket-based live cursors, shared analysis sessions, and group annotation support.

Adaptive Learning Paths

Use concept prerequisite graphs to generate personalised reading sequences. Recommend progression from simpler to more complex texts based on student level.

Fine-Tuned LLM

Fine-tune a domain-specific model on the CLEAR Corpus for even more accurate grade predictions and higher-quality rewriting.

Browser Extension

One-click readability analysis for any web page. Highlight difficult words in real-time as users browse articles and educational content.

Assessment Integration

Integration with LMS platforms (Moodle, Canvas) for automated reading-level checks on course materials and assignment submissions.
